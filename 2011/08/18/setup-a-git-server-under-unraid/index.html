Setup a Git Server under unRaid

2011-08-18
Computer
Linux · How-To · unRaid

The following tutorial will get you through the steps to install a fully functional Git server under unRaid.

Download and copy packages

wget http://natzo.com/packages/git-1.7.3.1-i686-1.txz /boot/packages

Setup global storage space for git repositories

  • Using unraid web interface, add a user share called “remote-backup”
  • Defines which drive(s) is(are) included.
  • Disable SMB and NFS mouting
  • Set split level to 999 for splitting all files across the disks

Create and install git user SSH key

If you have an exisiting private key on your remote PC just copy over its public key and rename it to

{% codeblock %} /boot/config/myconfig/ssh/id_git_rsa.pub {% endcodeblock %}

Install setup script

Git is configured with git-shell to prevent other uses besides git (i.e. ssh).

Save the following file on your flash drive.

{% codeblock /boot/config/myconfig/git-setup.sh lang:bash %} #!/bin/sh

#################################################################

MODIFY THE FOLLOWING VARIABLES ACCORDING TO YOUR INSTALLATION

## #################################################################

#Location of your binary packages on the USB Flash drive PKG_DIR=/boot/packages

#Location of your custom scripts and config files on the USB Flash drive #This scrip expects to find the following directoy within this one:

./ssh

CUSTOM_CONFIG_DIR=/boot/config/myconfig

This is the remote user that will have access to a chroot dir. You normally only have his public key

You need to have the following files under CUSTOM_CONFIG_DIR/ssh

REMOTE_USER=git

Directory under your unRAID user share that will host all the remote user’s backups

UNRAID_BACKUP_DIR=/mnt/user/git-repos GIT_GROUP=git

Expected files:

CUSTOM_CONFIG_DIR/ssh/

- id_git_rsa.pub

- sshd_config

- ssh_config

- ssh_host_dsa_key

- ssh_host_dsa_key.pub

- ssh_host_key

- ssh_host_key.pub

- ssh_host_rsa_key

- ssh_host_rsa_key.pub

- knwon_hosts????

- Optional: id_root_rsa.pub

##################################################################################

Installation & setup starts here, no need to modify anything beyond this point

##################################################################################

#########################

CHROOTED SFTP SETUP

#########################

#Package installation installpkg $PKG_DIR/git-1.7.3.1-i686-1.txz

#Add remote user groupadd $GIT_GROUP useradd -g $GIT_GROUP -s /usr/local/bin/git-shell -d /home/$REMOTE_USER -p ‘2234ZERZERZLREZOERZERZER’ $REMOTE_USER

mkdir -p /home/$REMOTE_USER/.ssh mkdir /git-repos

mount –bind $UNRAID_BACKUP_DIR /git-repos

cp –preserve=timestamps $CUSTOM_CONFIG_DIR/ssh/“id-”$REMOTE_USER”_rsa.pub” /home/$REMOTE_USER/.ssh/authorized_keys

chown $REMOTE_USER:$GIT_GROUP /home/$REMOTE_USER/.ssh /home/$REMOTE_USER/.ssh/* chmod 755 /home/$REMOTE_USER /home/$REMOTE_USER/.ssh chmod 600 /home/$REMOTE_USER/.ssh/* {% endcodeblock %}

Run the git install

Reboot or launch /boot/config/myconfig/git-setup.sh.

Create your first bare repo

unRAID will only contain “bare” GIT repositories that need to be first created on unRAID and not from the remote client.

# mkdir /git-repos/myrepo.git
# cd /git-repos/myrepo.git
# git --bare init
# chown -R git /git-repos/myrepo.git

Create your first working copy and push it to unRAID

On the remote client, you can use an existing repository or create a new one from scratch.

If you start a brand new project, create an empty dir and initialize the repository. If you have an exisiting git project you can skip the following steps and just cd into your existing project.

# cd myproject
# git init
# git add .
# git comment -a 'initial commit'

Then add the remote bare repository and push your initial commit.

# git remote add origin git@remotehost.com:/git-repos/myrepo.git
# git push origin master

Commit your changes and push them

Now work with git as usual, commit as usual and push your changes to unRAID when required.

# git commit -am 'my fix'
# git push master

Getting your repository on a new machine

# git clone git@remotehost.com:/git-repos/myrepo.git
..
# git commit -am 'my fix'
# git push master
